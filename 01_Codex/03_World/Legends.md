## Design Intent

Every legend contains truth.

Legends in Alamat should preserve memory, hide danger, explain taboos, guide travelers, protect sacred places, justify power, or distort history. A legend should never exist only as flavor text.

## Legend Rules

1. Every major legend must have a purpose in the living world.
2. Every legend should contain a truth, even if warped by time, politics, grief, fear, or humor.
3. Different regions may tell incompatible versions of the same legend.
4. Dungeons can reveal evidence behind legends without destroying their mystery.
5. Legends should affect behavior: routes, offerings, architecture, laws, festivals, jokes, or warnings.

## Legend Profile Template

| Field | Purpose |
| --- | --- |
| Legend Name | Working or approved title. |
| Region | Where the legend is told. |
| Common Version | What most people say happened. |
| Hidden Truth | What the legend preserves, distorts, or conceals. |
| Social Function | Warning, memory, law, grief, propaganda, navigation, ritual, taboo. |
| Related Location | Village, dungeon, ruin, route, shrine, island, or landmark. |
| Related Creature | Folklore being, animal, spirit, boss, or local omen. |
| Gameplay Use | Quest, puzzle, dungeon clue, secret, boss foreshadowing, item lore. |
| Canon Status | Draft, Review, Approved, Deprecated. |

## Legend Categories

| Category | Function |
| --- | --- |
| Origin Legend | Explains a place, people, island, ruin, or custom. |
| Warning Legend | Keeps people alive or obedient. |
| Travel Legend | Encodes route, tide, reef, mountain, or weather knowledge. |
| Creature Legend | Explains encounters, taboos, bargains, fear, or coexistence. |
| Ruin Legend | Preserves fragments of ancient history. |
| Family Legend | Protects lineage, shame, inheritance, or duty. |
| Political Legend | Justifies authority or hides a crime. |
