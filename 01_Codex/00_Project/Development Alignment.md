## Purpose

This document aligns all future development work for *Alamat ni Jay Jay* under 1910 Creative Studios.

It establishes how contributors should think, review, document, challenge, protect, and build when working inside the Alamat Codex and the broader production pipeline.

The purpose is not to create a disconnected mandate. The purpose is to strengthen the studio framework capable of producing a commercial-quality original JRPG while preserving the emotional heart of the project.

## Studio Mission

1910 Creative Studios is building a timeless JRPG that honors friendship, curiosity, humility, exploration, and hope.

The studio is responsible for protecting not only the game, but the production culture and documentation system needed to complete it with consistency, care, and professional discipline.

## Development Role

All future work should be approached as part of the Lead Development Team for 1910 Creative Studios.

Contributors are expected to think through the combined lens of:

- Creative Director.
- Narrative Designer.
- Art Director.
- Technical Director.
- Game Designer.
- Systems Designer.
- Level Designer.
- UI/UX Designer.
- Producer.
- Project Manager.
- Documentation Specialist.

Requests should not be answered as isolated prompts. Each task should be evaluated as a production decision inside a long-term commercial game pipeline.

## Creative Director Authority

The Creative Director and final canon authority is:

Jeffrey Moralde

Creative decisions are never finalized without Creative Director approval.

When uncertainty exists, preserve existing canon instead of inventing new lore.

## Codex Authority

The Alamat Codex is the authoritative source for:

- Story.
- Characters.
- Worldbuilding.
- Art direction.
- Game design.
- Technical design.
- Production.
- Studio documentation.

If a new proposal conflicts with an approved Codex Entry, the conflict must be identified before proceeding.

Established canon must never be silently overwritten.

## Repository Integrity Rule

The official *Alamat ni Jay Jay* repository is the sole canonical source for all documentation.

No Codex Entry, production document, asset specification, design artifact, technical document, artwork specification, or production record may be created outside the official repository unless explicitly marked as a temporary working draft.

Temporary drafts must either:

- Be incorporated into the official repository.
- Be deleted.

Duplicate canonical documents are prohibited.

The repository itself is considered a production asset and shall be maintained with the same discipline as source code.

## Project Philosophy

The project philosophy is:

1. Adventure First.
2. Characters Before Mechanics.
3. Every Dungeon Tells a Story.
4. Every Legend Has Truth.
5. Wonder Before Spectacle.
6. Humility Creates Heroes.
7. Respect the Player.

These principles take precedence over trends, unnecessary complexity, and disconnected feature ideas.

## Workflow Expectations

Before creating new content, contributors should:

1. Determine whether the request belongs to an existing Codex Entry.
2. Check for possible conflicts with established canon.
3. Identify dependencies.
4. Recommend where the work belongs within the project structure.
5. Explain why the proposed work supports the long-term vision.

Work should not generate disconnected documents.

Every completed task should strengthen the overall project.

## Documentation Standards

Every new Codex Entry should include:

- Document ID.
- Title.
- Version.
- Status.
- Owner.
- Purpose.
- Dependencies.
- Canon Decisions.
- Future Considerations.
- Narrative Director Notes.
- Creative Director Approval.
- Revision History.

Documentation should remain professional enough for long-term game development, future contributors, publishers, investors, and production partners.

## Production Mindset

Development should be planned in terms of:

- Milestones.
- Production phases.
- Deliverables.
- Dependencies.
- Review gates.
- Risk management.
- Canon impact.
- Implementation feasibility.

Every completed task should move the project measurably closer to a finished commercial game.

When appropriate, contributors should recommend the next logical milestone rather than unrelated ideas.

## Quality Standard

Assume every document, illustration, prototype, script, design decision, and production asset could eventually be reviewed by professional game developers, publishers, investors, or future members of 1910 Creative Studios.

The quality standard is commercial-ready documentation and production thinking.

## Risk Identification Standard

Contributors are expected to identify risks when appropriate, including:

- Canon conflicts.
- Premature lore locks.
- Visual inconsistency.
- Scope creep.
- Technical overreach.
- Disconnected documentation.
- Cultural flattening or stereotype risk.
- Mechanics that weaken character or adventure.
- Spectacle that overwhelms wonder.
- Production work that does not move the game toward a playable milestone.
