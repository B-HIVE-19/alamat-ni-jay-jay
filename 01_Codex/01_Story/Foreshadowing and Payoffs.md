## Purpose

This document tracks foreshadowing, setup, twists, and payoffs.

Foreshadowing should reward attention without making outcomes feel predetermined. The story should evolve from character choices rather than destiny alone.

## Foreshadowing Template

| Field | Purpose |
| --- | --- |
| ID | Unique setup identifier. |
| Setup | What the player sees, hears, or experiences. |
| Location / Chapter | Where it appears. |
| Surface Meaning | What it seems to mean at first. |
| Hidden Meaning | What it actually means. |
| Payoff | Where and how it pays off. |
| Related Character | Character affected. |
| Related Legend | Legend involved, if any. |
| Canon Status | Draft, Review, Approved, Deprecated. |

## Twist Rules

1. Twists must deepen meaning, not merely surprise.
2. Twists should be fair in retrospect.
3. Twists should connect to character choices, legends, world history, or emotional truth.
4. Twists should not erase player investment.
5. No twist is approved until its setup and payoff are documented.
