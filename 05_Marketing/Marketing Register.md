## Purpose

The Marketing Register organizes public-facing materials for *Alamat ni Jay Jay*.

Marketing materials must follow approved canon, the Project Charter, the Alamat Doctrine, the Art Bible, the Music Bible, and the emotional reason the project exists.

## Marketing Areas

| Area | Folder | Purpose |
| --- | --- | --- |
| Steam | `Steam/` | Store page drafts, capsules, tags, feature copy, build notes, and launch checklist. |
| Press Kit | `Press Kit/` | Studio description, game factsheet, approved screenshots, logos, contact info, and press-ready copy. |
| Trailer | `Trailer/` | Trailer concepts, scripts, shot lists, capture plans, edit notes, and export references. |
| Screenshots | `Screenshots/` | Approved screenshot candidates, capture notes, captions, and store/press selections. |
| Website | `Website/` | Website copy, page structure, media needs, announcements, and press links. |
| Social Media | `Social Media/` | Post drafts, content calendar, community guidelines, hashtags, and asset needs. |
| Logo | `Logo/` | Marketing-ready title treatments, studio marks, logo exports, and usage notes. |
| Box Art | `Box Art/` | Key art concepts, cover layouts, platform templates, and print/export planning. |

## Approval Rules

1. Marketing drafts are not canon.
2. Public claims must match approved Codex material.
3. Do not reveal story spoilers without explicit approval.
4. Do not overpromise features not represented in the Production Register or playable build.
5. Every public asset should feel unmistakably Alamat.
6. The tribute heart of the project must be protected in all public messaging.

## Memory Test

Before approving public-facing material, ask:

- Would this make 12-year-old Jeff excited to play?
- Would this help someone remember *Alamat ni Jay Jay* twenty years from now?
- Does this honor the emotional reason this project exists?

If any answer is no, keep designing.
