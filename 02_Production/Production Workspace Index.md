## Purpose

The Production Workspace Index organizes active working materials for *Alamat ni Jay Jay*.

These folders support day-to-day development. They are not canon by default. Canon changes must be reflected in the Alamat Codex and, when appropriate, recorded in the Decision Log.

## Workspace Areas

| Area | Folder | Purpose |
| --- | --- | --- |
| Current Sprint | `Current Sprint/` | Active sprint goals, priorities, blockers, and deliverables. |
| Meeting Notes | `Meeting Notes/` | Studio meeting notes, decisions to review, action items, and follow-ups. |
| Research | `Research/` | Cultural, design, technical, art, music, folklore, and market research notes. |
| Tasks | `Tasks/` | Task lists, work queues, assignments, and production checklists. |
| Character Reviews | `Character Reviews/` | Review notes for character bibles, locks, dialogue, visuals, and animation personality. |
| Story Reviews | `Story Reviews/` | Review notes for main story, side stories, chapters, themes, and emotional continuity. |
| Dungeon Reviews | `Dungeon Reviews/` | Review notes for dungeon purpose, layout, secrets, ecology, boss design, and memory target. |
| Playtest Notes | `Playtest Notes/` | Notes from prototype sessions, player reactions, confusion points, and improvement actions. |

## Production Rule

Working notes may be messy. Approved canon may not be messy.

When a working note creates or changes canon, move the approved result into the correct Codex document and record the decision when needed.

## Memory Test

Before closing a review or sprint item, ask:

- Would this make 12-year-old Jeff excited to keep playing?
- Would this make someone remember *Alamat ni Jay Jay* twenty years from now?
- Does this honor the emotional reason this project exists?

If any answer is no, keep designing.
